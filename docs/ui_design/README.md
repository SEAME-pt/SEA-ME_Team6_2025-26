<a id="sec-core-ideas"></a>
## üß† Core Ideas

- **User-Centered Design:** Focus on the needs and behaviors of users throughout the design process.
    - Involve users in testing and feedback to ensure the design meets their expectations.
- **Usability:** Ensure that the design is easy to use, intuitive and efficient.
- **Visual Hierarchy:** Organize elements to guide users attention toward the most important components.
- **Consistency:** Maintain a uniform look and feel accross all interfaces to foster familiarity.
    - This includes consistent use of colors, fonts and navigation elements.
- **Accessibility:** Design for all users, including those with disabilities, by considering various needs.
- **Feedback:** Provide clear feedback to users through notifications, animations or visual changes after actions.
- **Simplicity:** Strive for a clean and simples design that avoids unnecessary complexity.
- **Affordance:** Design elements should suggest their functionality, making it clear how to interact with them.

<a id="sec-design-goals"></a>
## üéØ Design Goals

Guiding principles that align the development of the instrumental cluster with the user needs, ensuring that it is not only functional but also appealing and intuitive.
- **Usability:**
    - The design should be intuitive, allowing users to quickly understand how to interact with the cluster. This includes minimizing learning time and reducing errors during use.
- **Aesthetics:**
    - The visual style should be modern and appealing, aligning with current automotive branding norms. A well-designed aesthetic enchances user engagement and satisfaction, making the experience more enjoyable.
- **Functionality:**
    - The instrumental cluster must effectively display critical information (speed, warning indicators). Functionality should be prioritized to ensure that users receive timely and relevant data while driving.
